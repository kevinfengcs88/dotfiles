# dotfiles
Due to the nature of my developer environment (WSL), these dotfiles are not all intended to be located at the Linux home directory or the `~` directory. As such, all folders and files are manually uploaded to this remote repository through the GitHub GUI rather than being tracked with Git in local repositories. Additionally, frequently updated configuration files, like those for the Neovim editor, are not guaranteed to be up-to-date to their dedicated repository counterparts.
